Application

This application is intened to ingest a shapefile representing glaciers and a Digital Elevation Model (DEM) and produce shapefiles ready to submit to both the Global Land Ice Measurements from Space (GLIMS) database and the Geophraphic Information Network of Alaska (GINA). Tables containing additional information can also be produced which contain statistics such as elevation, slope, aspect and area, both total and by elevation bin. These tables are joinable to the main GLIMS and GINA shapefiles by an ID fields located in all produces generated.

Application Parameters

Select Glaciers – This should be a shapefile contianing the extent of glaciers for a given period of time. Glaciers should already be diveded into drainage basins per the documentation provided by GLIMS () and any an all clean up should have already been completed. The file needs to be projected into GCS_WGS_1984 before the script is run and will be reporjected into Alaska Albers during the processing. This is a result of needing to do some calculations in decimal degrees and others in meters.

This shapefile should have the following attibute fields:

• ID – This serves as a 'primary key' field for the features within this layer. This field will be filled out during the process of running this script.

• Name (pre-populated) – The name field should contian the names of glaciers where applicable. This field should be filled out before running the script. Glaciers without names should be left blank.

• Date (pre-populated) – The date which best represents the feature mapped. For example, if a glacier was mapped from a Landsat image the date the image was acquired would be an approperiate date to include here.

• Area (pre-populated) – The field should contain the area of each glacier in square kilometers. This field should be pre populated before running the script.

• Code (pre-populated) – The 4 digit attribute field 'Code' is derived from the description given in Paul et. al. (2009) "Recommendations for the compilation of glacier inventory data from digital sources" and reproduce in the table below. For example, a calving tidewater glacier that should signs of surging would have a code of '0120'.

Select DEM – This should be a DEM that best represents the time period represented by the input glacier shapefile previously discussed. The DEM most fully cover the regions representing glaciers and be projected into Alaska Albers before it is used. The formate of the DEM doesn't matter as much either .tif or .img should work.

Select Output – This shoud simply be a folder you would like the results to be placed.
